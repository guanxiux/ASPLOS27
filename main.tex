\title{\sysname: Bridging Skip and Efficiency in GPU Sparse Convolution for 3D Point Cloud Processing}

\author{Anonymous}
\affiliation{\institution{Anonymous}}

\begin{abstract}
Sparse convolution is the dominant primitive for 3D point cloud perception,
yet existing GPU implementations face a fundamental tradeoff between
\emph{computation skipping} and \emph{kernel efficiency}.
Gather-GEMM-scatter approaches (e.g., MinkowskiEngine) skip empty neighbors
entirely but suffer from per-offset kernel launch overhead and load imbalance.
Implicit GEMM approaches (e.g., SpConv~v2, TorchSparse++) fuse all offsets
into a single efficient kernel but cannot skip empty neighbors, incurring
redundant computation proportional to sparsity.

We present \sysname, a sparse convolution runtime that eliminates this
tradeoff through two complementary execution models.
\emph{\skipgemm} precomputes all active neighbor pairs and executes them as
per-offset dense GEMM tiles within a single persistent-thread kernel,
achieving computation strictly proportional to active pairs.
\emph{\elasticgemm} dynamically redistributes idle threads within each
output-stationary thread block to cooperatively process different channel
segments of active points, trading a lightweight reduce for multiple
eliminated K-loop iterations.

On an RTX~2080~Ti with FP32 precision, \sysname's \skipgemm is
faster than all existing sparse convolution systems across the entire
active-ratio spectrum: $20\times$ faster than TorchSparse++ at 1\% active
ratio, and still $2\times$ faster at 30\%.
\end{abstract}
