\begin{abstract}
Sparse convolution is the dominant primitive for 3D point-cloud perception,
yet existing GPU runtimes face a fundamental tradeoff between
\emph{computation skipping} and \emph{execution efficiency}.
Gather-GEMM-scatter approaches skip empty neighbors but suffer from
per-offset launch overhead and load imbalance.
Implicit GEMM approaches fuse all offsets into one efficient kernel but
cannot skip empty neighbors, wasting compute proportional to sparsity.

We present \sysname, a sparse convolution runtime built on
\emph{worklist-driven execution}.
\sysname precomputes all active neighbor pairs, groups them by kernel
offset, and flattens the result into a contiguous tile stream executed by a
single persistent-thread kernel.
Empty offsets are skipped entirely; the stride loop provides automatic
global load balance; and each tile is a pure dense GEMM.
To recover data locality without heavy scheduling, \sysname applies
offset-major contiguous tile assignment and autotuned weight-stationary
micro-kernels.

We evaluate \sysname end-to-end on 3D object detection and semantic
segmentation models across real-world point-cloud datasets.
\sysname achieves \textbf{TODO}$\times$ end-to-end speedup over
TorchSparse++ on \textbf{TODO} and \textbf{TODO}$\times$ over SpConv~v2
on \textbf{TODO}, while producing bit-identical results under FP32.
\end{abstract}
